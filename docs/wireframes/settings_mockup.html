Mockup notes (not part of the real UI)

Settings is for things you set once and rarely touch. Filters that you adjust while browsing (salary minimum, employment type, job type, location) live on the Jobs page itself as filter chips, not here.

The "Hidden" sections (companies, title keywords, publishers) are framed as personal filtering preferences, not judgments about the listed entities. Each section has a "Clear all" button — useful when recording demos you don't want exclusion lists visible in.

Saved searches are queries that drive what JSearch fetches; they're applied at fetch time, not at view time. The "AI engineer" search above is paused (toggle off) — its results are skipped on the next refresh, but the search isn't deleted.

Contact info now has per-field "Show in letter" toggles with a live preview. Empty fields auto-disable their toggles. Name is always shown.

The PDF "drop zone" appears only when there's no current resume OR the user clicked "Replace" on the existing one. Hidden by default in this mockup.
