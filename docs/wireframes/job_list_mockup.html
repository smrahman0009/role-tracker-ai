Mockup notes (not part of the real UI)

Filter chips are indigo-tinted to signal they're "active constraints"; "Add filter" stays white-with-dashed-border so it reads as an action, not a filter.

All filter values become URL params (e.g. ?type=data-scientist&salary_min=80000), so refresh and back/forward work naturally.

The "result count" line ("Showing 3 of 7… 4 hidden by your filters") is important — never silently disappear data. Always show how many jobs the filters are hiding.

Job type and Location filters use a combobox (text input with autocomplete suggestions) rather than a closed dropdown — anything you type becomes a valid filter.
